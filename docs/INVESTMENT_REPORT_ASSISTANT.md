## 一、项目背景

### 1.1 业务场景

投资公司员工负责投前尽调和投后管理，需要定期汇报项目真实状况和风险点，供经理/老板做投资决策。

**核心痛点：**
- 员工汇报可能有意无意忽略关键信息
- 风险点描述模糊，缺乏量化
- 问题只摆在那里，没有解决方案
- 决策者难以快速获取关键信息

### 1.2 目标

通过 AI 多轮追问，挖掘项目真实状况和风险点，生成结构化报告，支持投资决策。

**核心价值：**
- 补齐员工忽略的关键信息
- 深挖风险根因和影响
- 推动形成解决方案
- 输出决策所需的关键信息

## 四、追问维度设计

### 4.1 投前尽调

**适用岗位：投资经理**

| 轮次 | 追问维度 | 具体问题 |
|------|----------|----------|
| 第1轮 | 项目基本面 | 项目/公司做什么？市场规模多大？<br>团队背景如何？核心人员履历？ |
| 第2轮 | 财务与业务 | 财务数据如何？营收/利润/现金流？<br>业务数据可验证吗？关键指标？ |
| 第3轮 | 风险识别 | 最大的风险是什么？<br>法律合规？政策风险？竞争风险？ |
| 第4轮 | 风险量化 | 风险发生的概率？影响程度？<br>有什么缓解措施？最坏情况？ |
| 第5轮 | 决策建议 | 你的判断：投/不投/观望？依据？<br>关键决策点？需要什么支持确认？ |

**适用岗位：法务**

| 轮次 | 追问维度 | 具体问题 |
|------|----------|----------|
| 第1轮 | 法律主体 | 公司架构？股权结构？历史沿革？ |
| 第2轮 | 合规情况 | 业务资质？牌照？合规风险？ |
| 第3轮 | 合同风险 | 重大合同？对赌协议？担保情况？ |
| 第4轮 | 诉讼风险 | 未决诉讼？潜在纠纷？ |
| 第5轮 | 法律意见 | 法律风险等级？是否建议投资？ |

**适用岗位：财务**

| 轮次 | 追问维度 | 具体问题 |
|------|----------|----------|
| 第1轮 | 财务概况 | 营收规模？利润情况？现金流？ |
| 第2轮 | 财务质量 | 收入确认方式？应收账款？存货？ |
| 第3轮 | 财务风险 | 负债情况？或有负债？税务风险？ |
| 第4轮 | 盈利预测 | 增长逻辑？预测依据？敏感性分析？ |

### 7.2 追问逻辑

```python
# 伪代码
async def process_message(session, user_message):
    """处理每一轮对话"""
    
    # 1. 添加用户消息到对话历史
    session.messages.append({"role": "user", "content": user_message})
    
    # 2. 判断当前轮次
    current_round = len([m for m in session.messages if m["role"] == "user"])
    
    # 3. 检查回答是否模糊
    if is_vague(user_message):
        # 追问细节，不进入下一轮
        response = await llm.ask_detail(user_message)
        session.messages.append({"role": "assistant", "content": response})
        return response
    
    # 4. 判断是否完成5轮
    if current_round < 5:
        # 继续下一轮追问
        questions = get_questions(session.type, session.position, current_round + 1)
        response = await llm.ask_next(session.messages, questions)
        session.messages.append({"role": "assistant", "content": response})
    else:
        # 生成报告
        report = await llm.generate_report(session.messages, session.report_template)
        session.report = report
        session.status = "已完成"
        
        # 保存并通知
        await save_to_bitable(session)
        await notify_manager(session.manager_id, report)
        
        response = "✅ 汇报已完成，感谢配合！"
    
    return response
```

```python
def is_vague(answer):
    """检测回答是否模糊"""
    # 1. 检查关键词
    for pattern in VAGUE_PATTERNS:
```

```python
SYSTEM_PROMPT = """
你是投资汇报助手。员工汇报工作后，你需要深入追问（最多5轮），目标是：

1. **挖出真实问题**
   - 不要接受模糊回答："有点困难" → "具体是什么困难？卡在哪里？"
   - 追问到底："为什么会出现这个问题？根本原因是什么？"

2. **识别风险和影响**
   - "这个问题会影响哪些方面？"
   - "如果不解决，最坏情况是什么？"
   - "风险等级：高/中/低？"

3. **推动形成解决方案**
   - "你打算怎么解决？有什么方案？"
   - "需要谁的支持？需要什么资源？"
   - "什么时候能解决？"

4. **确认责任人和时间**
   - "谁来负责跟进？"
   - "什么时候给反馈？"

**追问原则：**
- 每次只问1-2个问题，不要一次问太多
- 直接追问，不要客套
- 不接受"还好"、"差不多"这类模糊回答
- 直到问清楚：问题 → 原因 → 影响 → 方案 → 责任人 → 时间

**当信息完整后，输出结构化报告。**
"""
```
